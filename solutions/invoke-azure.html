oh! this solution of mine,

Invoke-AzureRmVMCommand

Get data from 700 servers in Azure cloud inside a closed network

  • Invoke-AzureRmVMCommand is used to trigger scripts locally on a target server
  • There is no way to GET data from different servers, which reside in different subscriptions

  1. I created a small Flask application and hosted it, once on a web server and once on Azure web app.
  2. The application simply takes the data from the GET URL and saves it to a CSV file.
  3. Then, using the above-mentioned cmdlet, I ran a script locally that collects the data and invokes a web request to my web API.
  4. This way I got data from our entire environment in just a matter of hours.

Resources would have been dedicated to this task and trained as it required some knowledge on Registry Entry. Also, because the server count is high, it would have taken at least 3 weeks to completely get the information. And not to ignore, human error, such as wrong or missing information.
